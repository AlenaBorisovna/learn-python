В схему таблицы в SQLite записывается описание всех других таблиц, индексов, триггеров и представлений, которые содержатся в базе данных.

Таблица схемы (sqlite_schema) содержит одну строку для каждой таблицы, индекса, представления и триггера в схеме. В ней есть следующие поля:

type. Содержит одну из текстовых строк: «table», «index», «view» или «trigger» в зависимости от типа объекта. Строка «table» используется для обычных и виртуальных таблиц.
name. Содержит название объекта.
tbl_name. Содержит название таблицы или представления, с которым связан объект. Для таблицы или представления это копия столбца name.
	Для индекса — название таблицы, которая внесена в указатель. Для триггера — название таблицы или представления, который заставляет триггер работать.
rootpage. Содержит номер страницы корня b-tree для таблиц и индексов. Для строк, которые определяют представления, триггеры и виртуальные таблицы, rootpage равен 0 или NULL.
sql. Содержит текст SQL, который описывает объект. Это инструкция CREATE TABLE, CREATE VIRTUAL TABLE, CREATE INDEX, CREATE VIEW или CREATE TRIGGER.
